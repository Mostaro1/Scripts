ProtectedAreas Class Documentation

This documentation provides an overview of the ProtectedAreas class and its methods in JavaScript.

Class: ProtectedAreas

The ProtectedAreas class is designed to manage protected areas and administrators for a world. It provides methods to create, modify, and delete protected areas, as well as manage whitelist and administrators.

Constructor


    constructor()

Initializes the ProtectedAreas class. If dynamic properties 'protectedAreas' and 'protectedAreasAdmins' do not exist in the world, they are initialized as empty arrays.

Methods

GET Methods

These methods allow you to retrieve information from the protected areas.

getProtectedAreas

    getProtectedAreas(): Array<Object>

Returns an array of protected areas as objects.

getArea

    getArea({ id: string }): Object | undefined

Returns a specific protected area by its unique ID. The 'id' parameter should be a string. Returns an object or undefined if not found.

getAreaWhitelist

    getAreaWhitelist({ id: string }): Array<string> | undefined

Returns the whitelist of players for a specific protected area. The 'id' parameter should be a string. Returns an array of strings or undefined if the area is not found.

getAdmins

    getAdmins(): Array<string>

Returns an array of administrators for the protected areas.

SET Methods

These methods allow you to add and set information for the protected areas.

areaWhitelistAdd

    areaWhitelistAdd({ id: string, player: string }): void

Adds a player to the whitelist of a specific protected area. The 'id' and 'player' parameters should be strings.

addAdmin

    addAdmin({ player: string }): void

Adds a player to the list of administrators for the protected areas. The 'player' parameter should be a string.

setArea

    setArea({ name: string, from: Vector, to: Vector }): void

Creates a new protected area with a unique name and specified boundaries. The 'name' parameter should be a string, and the 'from' and 'to' parameters should be Vector objects.

UPDATE Methods

These methods allow you to update information for the protected areas and administrators.

update

    update({ data: any }): void

Updates the protected areas data with the provided data. The 'data' parameter can be of any type.

updateAdmins

    updateAdmins({ data: any }): void

Updates the administrators data with the provided data. The 'data' parameter can be of any type.

updateAreaWhitelist

    updateAreaWhitelist({ id: string, data: any }): void

Updates the whitelist data for a specific protected area with the provided data. The 'id' parameter should be a string, and the 'data' parameter can be of any type.

DELETE Methods

These methods allow you to delete protected areas, administrators, and whitelist entries.

deleteArea

    deleteArea({ id: string }): void

Deletes a specific protected area by its unique ID. The 'id' parameter should be a string.

deleteAllAreas

    deleteAllAreas(): void

Deletes all protected areas.

clearAreaWhitelist

    clearAreaWhitelist({ id: string }): void

Clears the whitelist for a specific protected area. The 'id' parameter should be a string.

removeAdmin

    removeAdmin({ player: string }): void

Removes a player from the list of administrators. The 'player' parameter should be a string.

removeAllAdmins

    removeAllAdmins(): void

Clears the list of administrators.

areaWhitelistRemove

    areaWhitelistRemove({ id: string, player: string }): void

Removes a player from the whitelist of a specific protected area. The 'id' and 'player' parameters should be strings.

This documentation outlines the functionality provided by the ProtectedAreas class for managing protected areas and administrators in a world.
